# Voice Assistant Knowledge Base

## Overview

The Voice Assistant API exposes a collection of endpoints that combine speech technologies, email automation, and LLM-powered chat. The Retrieval-Augmented Generation (RAG) endpoint uses these notes to ground its responses.

## Available Endpoints

- `POST /api/chat` – Plain LLM chat with a concise assistant persona.
- `POST /api/rag` – Answers questions grounded in the knowledge base.
- `POST /api/stt` – Transcribes audio uploads into text.
- `POST /api/tts` – Synthesizes speech (MP3) from text prompts.
- `POST /api/email` – Sends transactional emails through the configured SMTP account.

## RAG Behavior

1. Documents are loaded from the `knowledge_base/` directory (Markdown and text files).
2. Each document is chunked into ~1,200 character blocks and embedded with the `nomic-embed-text` model through Ollama.
3. The top `RAG_TOP_K` chunks above the `RAG_MIN_SCORE` threshold are included as context for the LLM.
4. The assistant only answers with the provided context and reports if information is missing.
